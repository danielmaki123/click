# Marco Operativo (Template)

Contenido obligatorio:
1. Parametros del proyecto.
2. Modo de operacion (DUAL_IDE por defecto).
3. Reglas no negociables.
4. Gates (bloque + integracion).
5. Politica de integracion sin ruptura.
6. Protocolo de comunicacion eficiente.
7. Orden y optimizacion documental.
8. Gate pre-decision del supervisor.
9. Decision log oficial del supervisor.
10. Recomendacion de inicializacion Git (baseline de control).

Seccion obligatoria de orden:
1. Estructura oficial de carpetas.
2. Dueño de orden del repositorio.
3. Frecuencia de limpieza.
4. Criterios para archivar o eliminar temporales.

Seccion obligatoria de decisiones:
1. Checklist pre-decision.
2. Referencias que el supervisor debe consultar.
3. Formato de veredicto.
4. Regla de registro en decision log.

Seccion recomendada de inicializacion:
1. Validar existencia de repositorio Git en raiz.
2. Inicializar Git y .gitignore si no existe.
3. Crear commit baseline inicial del proyecto.
4. Confirmar arbol limpio antes de empezar bloques de ejecucion.
