# Plan de Implementación - Fase 3 - P1: Webhook ManyChat

Este documento define el contrato y alcance técnico para la integración del webhook de ManyChat con el backend de Click.

## Alcance Técnico
- Implementación de endpoint seguro para recibir acciones de ManyChat (ACCEPT, REJECT, READY, COMPLETE).
- Validación de autenticidad mediante secret compartido en el body.
- Actualización de estados en la base de datos PostgreSQL.
- Registro de timestamps críticos (`acceptedAt`, `readyAt`).

## Contrato de API (Congelado)
- **Endpoint**: `POST /api/webhooks/manychat`
- **Seguridad**: Campo `secret` validado contra `MANYCHAT_WEBHOOK_SECRET`.
- **Payload**:
```json
{
  "orderId": "uuid",
  "action": "ACCEPT | REJECT | READY | COMPLETE",
  "secret": "string"
}
```

## Mapeo de Estados
- `ACCEPT` -> `ACCEPTED`
- `REJECT` -> `REJECTED`
- `READY` -> `READY`
- `COMPLETE` -> `COMPLETED`
